## ğŸ” Authentication Flow

1. User selects role (User or Provider)
2. Clicks "Login with Google"
3. Google OAuth authentication
4. Backend receives:
   - google_id
   - name
   - email
   - role
5. Backend creates/updates user record
6. Generates JWT-style token
7. Returns token + user data
8. Flutter stores token in SharedPreferences
9. All API calls include token in Authorization header

## ğŸ”’ Security Features

1. **Token-based Authentication**
   - JWT-style tokens
   - Stored securely in SharedPreferences
   - Required for all API calls

2. **Role-based Access Control**
   - Middleware validates user roles
   - Users can only access user endpoints
   - Providers can only access provider endpoints

3. **SQL Injection Prevention**
   - PDO prepared statements
   - Parameter binding

4. **CORS Protection**
   - Configured headers
   - Origin validation

5. **XSS Prevention**
   - Input sanitization
   - Output encoding
